# 高级语言程序设计大作业 - CGame游戏项目报告

## 一. 项目基本信息

**项目名称**: 幸存者大作战 **项目类型**: 像素平台游戏
**开发语言**: C++ 17
**开发框架**: Qt 6.9.1
**开发环境**: Windows 11, Qt Creator, MinGW GCC
**项目成员**: 个人项目

## 二. 项目设计思路

### 1. 游戏模式设计

本项目设计了两种游戏模式：

**单人模式**：采用"幸存者"类游戏玩法，玩家控制角色在地图上移动，击败不断生成的敌人，获取经验并升级。升级时可以选择强化能力，如增加攻击力、提升速度、减少技能冷却等。游戏目标是在敌人的围攻下生存尽可能长的时间。

**多人模式**：支持2-4人对抗，玩家之间可以相互攻击，最后存活的玩家获胜。多人模式下玩家拥有不同的控制方式，增加了游戏的趣味性和竞争性。

### 2. 核心系统设计

**玩家系统**：

- 实现了多个可选择的角色，每个角色有不同的外观
- 单人模式下实现了技能系统，包括导弹雨、能量护盾、闪电链三种技能
- 多人模式下实现了基本的移动、攻击、跳跃等操作

**敌人系统**：

- 单人模式下定期生成敌人，敌人会自动向玩家移动
- 实现了普通敌人和BOSS，BOSS每3级出现一次，拥有更高的生命值和伤害
- 敌人死亡时会掉落经验球，被玩家拾取后增加经验值

**升级系统**：

- 玩家获得足够经验后升级，显示升级面板
- 每次升级提供三个随机的升级选项，包括攻击力、速度、生命值、攻击速度、技能伤害、技能冷却等
- 升级选项在进入升级页面时随机生成，而不是在选择时变化

**UI系统**：

- 实现了主菜单、玩家数量选择、角色选择、游戏界面、升级面板、数据面板、结算面板等UI界面
- 界面设计简洁明了，操作直观

### 3. 技术架构设计

- **核心类**：CGame类作为游戏的主类，负责游戏的初始化、更新、绘制等核心功能
- **游戏循环**：使用QTimer实现游戏主循环，以固定时间步长更新游戏逻辑
- **事件处理**：使用Qt的事件系统处理键盘输入、鼠标点击等事件
- **绘制系统**：使用QPainter在paintEvent中绘制游戏场景
- **资源管理**：使用Qt的资源系统管理图片、音效等资源

## 三. 项目实现细节

### 1. 游戏主循环

游戏主循环使用QTimer实现，以16ms为间隔（约60FPS）触发游戏逻辑更新和界面重绘。为了确保游戏在不同性能的设备上运行速度一致，实现了固定时间步长的游戏更新机制，使用QElapsedTimer和accumulator来控制游戏速度。

### 2. 玩家控制

**单人模式**：

- 使用WASD键控制移动
- 空格键攻击
- F键释放技能
- P键打开数据面板

**多人模式**：

- 玩家1：使用WASD键控制
- 玩家2：使用方向键控制
- 玩家3：使用JKLI键控制
- 玩家4：使用1352键控制

### 3. 敌人

敌人使用简单的追逐算法，向玩家位置移动。在单人模式下，敌人会定期生成，数量随玩家等级增加而增加。BOSS每3级出现一次，拥有更高的生命值和伤害。

### 4. 碰撞检测

实现了多种碰撞检测：

- 玩家与敌人的碰撞检测
- 子弹与敌人的碰撞检测
- 玩家与经验球的碰撞检测
- 玩家与道具的碰撞检测
- 玩家与地形的碰撞检测（多人模式）

### 5. 技能系统

实现了三种技能：

- **导弹雨**：发射10枚追踪导弹，对敌人造成大量伤害
- **能量护盾**：恢复玩家50点生命值
- **闪电链**：攻击多个敌人，最多链3个敌人，并使敌人眩晕1秒

### 6. 升级系统

每次升级时，游戏会随机生成三个升级选项，玩家可以选择其中一个进行强化。升级选项包括：

- 攻击力 +5
- 速度 +1
- 生命值 +20
- 攻击速度 +5%
- 技能伤害 +50
- 技能冷却 -10%

### 7. 数据面板

按P键可以打开数据面板，显示玩家的详细属性：

- 攻击力
- 攻击速度
- 当前技能
- 技能伤害
- 技能冷却
- 生命值
- 速度
- 等级
- 经验
- 击杀数

## 四. 遇到的问题与解决方案

1. **问题**：多人模式跳跃一次落地后，不能再起跳了
   **解决方案**：更新checkCollisions函数，在玩家落地时重置跳跃状态和跳跃次数
2. **问题**：追踪子弹指针访问违规
   **解决方案**：在checkBulletCollisions函数中添加try-catch块和额外的指针有效性检查，确保指针安全
3. **问题**：按右方向键时游戏帧率加快
   **解决方案**：实现了固定时间步长的游戏更新机制，使用QElapsedTimer和accumulator来控制游戏速度
4. **问题**：单人模式的怪物出现在多人模式中
   **解决方案**：修改updateGame函数，只在单人模式下生成和更新敌人；修改checkBulletCollisions函数，只在单人模式下检测敌人碰撞；修改drawGame函数，只在单人模式下渲染敌人
5. **问题**：多人模式下生命值不会改变
   **解决方案**：在checkBulletCollisions函数中添加多人模式下的敌人与玩家碰撞检测逻辑
6. **问题**：多人模式下攻击冷却时间过短
   **解决方案**：为多人模式下的所有玩家设置了0.8秒的攻击冷却时间

## 五. 收获与总结

通过本次大作业，我获得了以下收获：

1. **深入理解了Qt框架**：对Qt的核心机制，如事件循环、绘图系统、资源管理、事件处理等有了更深入的理解和实践经验
2. **掌握了C++游戏开发基础**：学习了如何运用面向对象思想设计游戏实体类，管理游戏状态，实现基本的游戏逻辑（移动、碰撞、AI、升级系统）
3. **熟悉了游戏开发流程**：从游戏设计、资源准备、代码实现到测试调试，完整地体验了游戏开发的全过程
4. **提升了调试能力**：在解决各种问题时，学会了通过日志分析、断点调试、隔离问题等方法定位和修复Bug，提升了调试复杂程序的能力
5. **理解了游戏物理和逻辑**：实现了固定时间步长的游戏更新机制，确保游戏在不同设备上运行速度一致
6. **认识到代码组织的重要性**：合理的代码结构和命名规范对于大型项目的可维护性至关重要
7. **实践了UI与逻辑分离**：通过将游戏逻辑和UI绘制分离，提高了代码的可读性和可维护性

**总结**：本项目成功实现了一个具有单人模式和多人模式的像素平台游戏，涵盖了图形界面、事件处理、游戏逻辑、资源管理等多个方面。开发过程充满挑战，但也极大地锻炼了我的C++编程能力、Qt应用能力和问题解决能力。虽然仍有一些待完善之处（如更丰富的敌人类型、更复杂的升级系统、性能优化等），但基本达到了预期的目标。
